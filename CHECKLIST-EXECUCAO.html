Fase 1 Site + Domínio no ar ~1 tarde · Resultado: site publicado
Verificar disponibilidade do domínio
Acesse namecheap.com e pesquise "duploclique.fm". Se não disponível, tente: duploclique.com.br (Registro.br), duplo-clique.fm, ou duploclique.fm.
→ namecheap.com
Registrar o domínio .fm
Comprar "duploclique.fm" no Namecheap. Pague por 2 anos se possível (desconto + proteção de marca). Ative "WhoisGuard" para privacidade.
Custo: ~US$30/ano · Cartão de crédito internacional
Criar conta no GitHub e subir os arquivos do site
Criar repositório público "duplo-clique-site". Fazer upload da pasta /site/ que está nos seus arquivos: index.html + pasta data/ com episodes.json + pasta automation/.
→ github.com
Os arquivos estão prontos na pasta "site" que gerei para você. Basta fazer upload.
Criar conta no Vercel e conectar ao GitHub
Acesse vercel.com, crie conta com sua conta do GitHub, clique em "Add New Project", selecione o repositório "duplo-clique-site". O Vercel detecta HTML automaticamente — clique em Deploy.
→ vercel.com
O primeiro deploy leva ~1 minuto. Você vai receber uma URL do tipo "duplo-clique-site.vercel.app" — é onde o site estará temporariamente até o domínio propagar.
Apontar o domínio para o Vercel
No Vercel: Project Settings → Domains → Add "duploclique.fm". O Vercel mostra os nameservers. No Namecheap: Domain → Manage → Nameservers → copie os do Vercel.
A propagação de DNS leva entre 30 minutos e 24 horas. Normal — não se preocupe se não funcionar imediatamente.
Popular os episódios reais no episodes.json
O arquivo já tem 7 episódios com dados reais que conseguimos levantar. Revise os títulos, datas e URLs do Spotify/YouTube. Edite direto no GitHub (botão "Edit" no arquivo) e faça commit — o Vercel atualiza automaticamente.
Prioridade máxima: adicionar as URLs reais do YouTube e Spotify para cada episódio. Os timestamps podem ser ajustados depois quando a automação rodar.
Criar conta no Beehiiv e configurar a newsletter
Criar conta em beehiiv.com (plano Launch, grátis). Configurar: nome da publicação "Duplo Clique", logo, cor primária #1B7FE0. Ir em Settings → API → Generate API Key (salvar em lugar seguro).
→ beehiiv.com
Conectar o formulário do site ao Beehiiv
No Beehiiv: Forms → Create Form → copie o embed code ou use o endpoint da API. No arquivo index.html do site, substitua o formulário atual pelo endpoint do Beehiiv.
O Beehiiv fornece um endpoint de API para adicionar assinantes via POST. Isso pode ser configurado direto no JavaScript do site — eu posso gerar esse código quando você tiver a API key.
Fase 2 Automação completa ~1 dia · Resultado: pipeline rodando sozinho
Criar conta no Make.com (plano Core)
Criar conta em make.com. Escolher plano Core (US$9/mês) — necessário para mais de 1.000 operações mensais e para usar módulos premium como GitHub e YouTube.
→ make.com
Criar conta Anthropic e gerar API Key
Acessar console.anthropic.com, criar conta, ir em API Keys → Create Key. Adicionar crédito de US$5 (dura o ano todo para o volume do podcast).
→ console.anthropic.com
Guarde a API key — aparece só uma vez. Formato: sk-ant-api03-...
Gerar GitHub Personal Access Token
No GitHub: Settings → Developer Settings → Personal Access Tokens → Tokens (classic) → Generate. Permissões necessárias: "repo" (full control). O Make.com vai usar esse token para fazer commits.
Guarde o token — aparece só uma vez. Formato: ghp_...
Configurar o cenário no Make.com
Seguir o guia em automation/make-scenario-guide.html. Conectar: RSS (Buzzsprout) → Buzzsprout API → Claude API → JSON Parser → GitHub Get → GitHub Update + Beehiiv Create Post + (YouTube opcional primeiro).
O guia está detalhado módulo por módulo. Reserve ~3 horas para a primeira configuração. Depois de configurado, não precisa tocar mais.
Testar o pipeline com episódio existente
No Make.com, use "Run once" apontando para um episódio já publicado. Verificar: JSON do Claude válido? Site atualizado? Rascunho no Beehiiv criado? Ajustar o prompt se necessário.
Faça pelo menos 3 testes com episódios diferentes antes de ativar o cenário. O prompt do Claude pode precisar de ajustes para o estilo do Duplo Clique.
Configurar YouTube Data API
No Google Cloud Console: criar projeto → APIs → YouTube Data API v3 → Enable → Credentials → OAuth 2.0 Client ID → autorizar conta do canal. No Make.com: conectar YouTube com essas credenciais.
→ Google Cloud Console
Essa é a etapa mais complexa tecnicamente. Se travar, pode deixar para a Fase 3 — o restante do pipeline funciona sem ela.
Enviar a primeira newsletter pela automação
Publicar um episódio novo no Buzzsprout normalmente. Aguardar até 15 minutos. Verificar se o Make.com rodou, o rascunho está no Beehiiv. Revisar e enviar.
Esta é a validação final da Fase 2. Se chegou até aqui e funcionou, o sistema está operacional.
Fase 3 Arquivo histórico + SEO ~1 fim de semana · Resultado: biblioteca completa
Rodar o pipeline retroativamente em todos os episódios
No Make.com, rodar o cenário manualmente para cada episódio anterior — basta passar a URL do episódio no Buzzsprout. O Claude vai gerar os segmentos para o arquivo histórico.
Revisar e publicar os episodes.json do arquivo completo
Depois do pipeline retroativo, o episodes.json terá todos os episódios com segmentos. Revisar os mais importantes (últimos 10) antes de publicar o arquivo completo no site.
Adicionar chapters nos vídeos do YouTube antigos
Para os episódios já publicados no YouTube, usar o Make.com ou adicionar manualmente os chapters gerados pelo Claude. Priorize os 10 episódios mais acessados.
Configurar Google Analytics e Search Console
Adicionar o tag do Google Analytics 4 no index.html. Registrar o domínio no Google Search Console e enviar o sitemap para indexação.
Eu posso gerar o sitemap.xml automático baseado no episodes.json quando chegar nesta etapa.
Registrar também duploclique.com.br (opcional mas recomendado)
No Registro.br: registrar "duploclique.com.br" e configurar redirect para "duploclique.fm". Protege a marca no Brasil.
→ registro.br
Custo: ~R$40/ano
