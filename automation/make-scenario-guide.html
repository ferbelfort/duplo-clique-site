Make.com — Guia de Configuração

Passo a passo para montar o pipeline de automação completo: Buzzsprout → Claude → GitHub + Beehiiv + YouTube

Contas que você precisa criar (antes de tudo)

Make.com
make.com
Plano Core, US$9/mês. É o hub central de toda automação.
Anthropic Console
console.anthropic.com
Criar conta, gerar API Key, adicionar crédito (US$5 dura o ano todo).
Beehiiv
beehiiv.com
Plano Launch (grátis). Gerar API Key em Settings → API.
GitHub
github.com
Criar repositório "duplo-clique-site" (público ou privado). Gerar Personal Access Token em Settings → Developer.

Visão geral do cenário Make.com

Um único cenário com 9 módulos em sequência, com 3 saídas paralelas no final:

📡
1. RSS · Watch Feed
Monitora o feed RSS do Buzzsprout. Dispara quando detecta novo episódio.
🎙
2. HTTP · Get Buzzsprout Transcription
Chamada GET à API do Buzzsprout para buscar a transcrição do episódio recém-publicado.
🤖
3. Anthropic Claude · Create Message
Envia transcrição + título ao Claude com o prompt de segmentação. Recebe JSON estruturado.
🔧
4. JSON · Parse
Faz parse do JSON retornado pelo Claude em objeto utilizável pelos próximos módulos.
📂
5. GitHub · Get File (episodes.json)
Busca o arquivo episodes.json atual do repositório para poder atualizar sem sobrescrever os episódios anteriores.
⚡
6. Tools · Set Variable (merge JSON)
Combina o novo episódio (do Claude) com o array existente de episódios do GitHub, criando o JSON atualizado.

A partir daqui, 3 saídas rodam em paralelo (roteador):

🌐 Site
GitHub · Update File — faz commit do episodes.json atualizado. Vercel refaz o deploy automaticamente em ~30s.
📧 Newsletter
Beehiiv · Create Post — cria rascunho da newsletter já formatada com intro + blocos por tema + CTA.
▶️ YouTube
YouTube · Update Video — adiciona chapters formatados na descrição do vídeo usando os timestamps do Claude.
🔔
10. Notificação final (e-mail ou WhatsApp)
Envia resumo: "Episódio #XX processado ✅ — Site atualizado, rascunho no Beehiiv, chapters no YouTube."

Configuração módulo por módulo

1
RSS · Watch Feed Items

No Make.com, adicione o módulo "RSS" → "Watch RSS Feed Items".

URL do feed: https://feeds.buzzsprout.com/2437885.rss Intervalo de verificação: 15 minutos
O Make.com vai verificar o feed a cada 15 minutos. Quando detectar um novo item (novo episódio), dispara o cenário automaticamente.
2
HTTP · Get Buzzsprout Transcription

Adicione módulo "HTTP" → "Make a Request". Configure assim:

Method: GET URL: https://www.buzzsprout.com/api/{{seu_account_id}}/episodes/{{episode_id}}/transcript.txt Headers: Authorization: Token token={{sua_api_key_buzzsprout}} Onde {{episode_id}} vem do campo "guid" ou "id" do módulo RSS anterior.
Sua API key do Buzzsprout fica em: My Account → API Access. Seu account ID aparece na URL quando você está logado no Buzzsprout (ex: buzzsprout.com/2437885 → ID é 2437885).
3
Anthropic Claude · Create Message

Adicione módulo "Anthropic Claude" → "Create a Message". Conecte sua API key.

Model: claude-sonnet-4-5 Max Tokens: 2000 Temperature: 0.3 System: [cole aqui o conteúdo completo do arquivo claude-prompt.txt] User: EPISÓDIO: {{title}} (do módulo RSS) NÚMERO: {{extrair do title com regex, ex: #41}} DURAÇÃO: {{itunes:duration}} (do módulo RSS) DATA: {{pubDate}} (do módulo RSS) TRANSCRIÇÃO: {{Data do módulo HTTP anterior}}
A transcrição pode ser longa. Verifique se o Make.com não está truncando o campo. Se precisar, use o módulo "Text aggregator" antes para garantir que o texto completo está passando.
4
JSON · Parse JSON

Adicione módulo "JSON" → "Parse JSON". No campo "JSON String", selecione o output do Claude (campo "content[].text").

Isso transforma a string JSON do Claude em um objeto Make.com com campos acessíveis nos próximos módulos. Se o Claude retornar texto antes do JSON (o que não deve acontecer com temperature 0.3), adicione um módulo "Text" → "Replace" para remover qualquer texto antes do primeiro "{".
5
GitHub · Get a File

Adicione módulo "GitHub" → "Get a File". Conecte sua conta GitHub.

Repository: duplo-clique-site File path: data/episodes.json Branch: main
6
Tools · Set Variable (merge)

Adicione módulo "Tools" → "Set Variable". Aqui você combina o novo episódio com o array existente.

Variable name: updated_episodes_json Variable value: (use a função JSON.stringify do Make para combinar arrays) Fórmula aproximada: { "episodes": [ {{JSON do novo episódio (do Claude)}}, ...{{episódios existentes do GitHub}} ] }
Esta é a etapa mais técnica. Se precisar de ajuda, no Make.com você pode usar a função "parseJSON" e "toString" para manipular os arrays. Alternativamente, use um módulo "HTTP" apontando para um webhook que você cria — é possível simplificar com um script Python no Vercel como API route.
7A
GitHub · Update a File (site)
Repository: duplo-clique-site File path: data/episodes.json Content: {{updated_episodes_json}} Commit message: "Add episode #{{id}} — {{title}}" Branch: main
O Vercel detecta o commit e refaz o deploy automaticamente em ~30 segundos. O site atualiza sozinho.
7B
Beehiiv · Create Post (newsletter draft)
Publication ID: (seu ID do Beehiiv, em Settings → API) Title: {{newsletter.subject}} (do JSON do Claude) Subtitle: {{newsletter.preview_text}} Status: draft (para você revisar antes de enviar) Body HTML: <p>{{newsletter.intro}}</p> {{para cada segmento em segments}}: <div> <span>{{segment.theme_label}}</span> <h2>{{segment.title}}</h2> <p>{{segment.description}}</p> <p><strong>{{segment.hook}}</strong></p> </div> <a href="{{youtube_url}}">{{newsletter.cta_text}}</a>
Com status "draft", o Beehiiv não envia automaticamente. Você recebe a notificação, abre o Beehiiv, revisa em 2 minutos e clica em "Send". Quando estiver confiante na qualidade, mude para "confirmed" para envio automático.
7C
YouTube · Update Video (chapters)
Video ID: (precisa mapear episódio → video ID) Use uma coluna no episodes.json para armazenar o youtube_video_id Description: (descrição atual do vídeo) + "\n\n" + {{youtube_chapters do Claude}} Onde youtube_chapters tem o formato: 0:00 Introdução 12:30 CRISPR e edição genética 31:15 AlphaFold 48:00 Reflexões finais
O YouTube Data API precisa de autorização OAuth da conta do canal. Configure em Google Cloud Console → Credentials → OAuth 2.0. No Make.com, use "YouTube" → "Update a Video". O Video ID do episódio precisa ser adicionado manualmente no episodes.json quando o vídeo for publicado (ou usar YouTube Search API para encontrar pelo título — mais complexo).
8
Email / WhatsApp · Notificação final

Use o módulo "Email" (Make.com tem SMTP integrado) ou "WhatsApp Business" via Zapi.com.br.

Para: fernanda@email.com Assunto: ✅ Duplo Clique #{{id}} processado Corpo: - Site: atualizado com {{segments.length}} segmentos - Newsletter: rascunho criado no Beehiiv — revisar e enviar - YouTube: chapters adicionados - Custo Claude API: ~US$0.08 Link do rascunho no Beehiiv: {{beehiiv_post_url}}

Como testar antes de ativar

  • No Make.com, use "Run once" apontando para um episódio já publicado (não o mais recente, para não criar duplicata)
  • Verifique o output do Claude — os segmentos fazem sentido? O JSON é válido?
  • Verifique o arquivo episodes.json no GitHub — o novo episódio foi adicionado corretamente?
  • Verifique o deploy no Vercel — o site mostra o episódio novo?
  • Verifique o rascunho no Beehiiv — o HTML está formatado corretamente?
  • Verifique o vídeo no YouTube — os chapters aparecem na descrição?
  • Se tudo OK: ative o cenário. Daqui em diante, cada publicação no Buzzsprout dispara tudo automaticamente.

Iterando o prompt do Claude

Nas primeiras rodadas, compare o output do Claude com a sua percepção do episódio. O prompt pode ser ajustado para:

  • Segmentos muito curtos? Ajuste "mínimo de 8 minutos" para 10 ou 12 no prompt
  • Temas classificados errado? Adicione exemplos no prompt ("quando houver discussão de mercados financeiros, use 'economia' não 'geopolitica'")
  • Hook muito genérico? Adicione instrução: "o hook deve mencionar um ângulo específico, não apenas o tema"
  • Intro da newsletter muito longa? Limite explícito: "máximo 80 palavras para a intro"
